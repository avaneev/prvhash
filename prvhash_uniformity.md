## The PRVHASH Core64 Function

```c
static uint64_t prvhash_core64( uint64_t* const Seed0,
    uint64_t* const lcg0, uint64_t* const Hash0 )
{
    uint64_t Seed = *Seed0;
    uint64_t lcg = *lcg0;
    uint64_t Hash = *Hash0;

    Seed *= lcg * 2 + 1;                          // Step 1
    const uint64_t rs = Seed >> 32 | Seed << 32;  // Step 2 (rotate)
    Hash += rs + 0xAAAAAAAAAAAAAAAA;              // Step 3
    lcg += Seed + 0x5555555555555555;             // Step 4
    Seed ^= Hash;                                 // Step 5
    const uint64_t out = lcg ^ rs;                // Step 6

    *Seed0 = Seed; *lcg0 = lcg; *Hash0 = Hash;
    return out;
}
```

## Part 1: Joint Uniformity via Bijectivity (The Foundation)

**Theorem 1.1 (State Transition is a Permutation):** The PRVHASH state update function $F: (\mathbb{Z}/2^{64}\mathbb{Z})^3 \to (\mathbb{Z}/2^{64}\mathbb{Z})^3$ defined by $F(\text{Seed}, \text{lcg}, \text{Hash}) = (\text{Seed}', \text{lcg}', \text{Hash}')$ is a bijection.

**Proof:** We construct the explicit inverse $F^{-1}$.

Given the output state $(\text{Seed}', \text{lcg}', \text{Hash}')$, we recover the input state as follows:

1. **Recover $\text{Seed}_{\text{new}}$:** From Step 5, $\text{Seed}' = \text{Seed}_{\text{new}} \oplus \text{Hash}'$. Therefore:
   $$\text{Seed}_{\text{new}} = \text{Seed}' \oplus \text{Hash}'$$

2. **Recover $rs$:** From Step 2, $rs = \text{rot32}(\text{Seed}_{\text{new}})$. Since a 32-bit rotation on a 64-bit word is self-inverse ($\text{rot32}(\text{rot32}(x)) = x$), we have:
   $$rs = \text{rot32}(\text{Seed}' \oplus \text{Hash}')$$

3. **Recover $\text{Hash}$:** From Step 3, $\text{Hash}' = \text{Hash} + rs + 0x\text{AA...AA}$. Therefore:
   $$\text{Hash} = \text{Hash}' - rs - 0x\text{AA...AA}$$

4. **Recover $\text{lcg}$:** From Step 4, $\text{lcg}' = \text{lcg} + \text{Seed}_{\text{new}} + 0x\text{55...55}$. Therefore:
   $$\text{lcg} = \text{lcg}' - \text{Seed}_{\text{new}} - 0x\text{55...55}$$

5. **Recover $\text{Seed}$:** From Step 1, $\text{Seed}_{\text{new}} = \text{Seed} \cdot (2 \cdot \text{lcg} + 1)$. Since $2 \cdot \text{lcg} + 1$ is always odd (hence a unit in $\mathbb{Z}/2^{64}\mathbb{Z}$), it has a multiplicative inverse, which is evaluated using the *already recovered* $\text{lcg}$ from Step 4. Therefore:
   $$\text{Seed} = \text{Seed}_{\text{new}} \cdot (2 \cdot \text{lcg} + 1)^{-1} \pmod{2^{64}}$$

Each step is a well-defined deterministic function. The composition yields $F^{-1}: (\mathbb{Z}/2^{64}\mathbb{Z})^3 \to (\mathbb{Z}/2^{64}\mathbb{Z})^3$. Since $F^{-1}$ exists, $F$ is bijective. ∎

---

**Theorem 1.2 (Conservation of Uniformity):** If the initial state $(\text{Seed}_0, \text{lcg}_0, \text{Hash}_0)$ is uniformly distributed on $(\mathbb{Z}/2^{64}\mathbb{Z})^3$, then after any number of iterations, the state $(\text{Seed}_n, \text{lcg}_n, \text{Hash}_n)$ remains uniformly distributed on $(\mathbb{Z}/2^{64}\mathbb{Z})^3$.

**Proof:** Let $X = (\text{Seed}, \text{lcg}, \text{Hash})$ be uniformly distributed on the finite set $S = (\mathbb{Z}/2^{64}\mathbb{Z})^3$. For any $y \in S$:

$$\Pr[F(X) = y] = \Pr[X = F^{-1}(y)] = \frac{1}{|S|}$$

The first equality holds because $F$ is a function; the second holds because $X$ is uniform and $F^{-1}(y)$ is a unique element of $S$. Therefore $F(X)$ is uniform on $S$. By induction, $F^n(X)$ is uniform for all $n \geq 0$. ∎

## Part 2: Marginal Uniformity (Corollaries)

**Corollary 2.1:** $\text{Seed}'$, $\text{lcg}'$, and $\text{Hash}'$ are each marginally uniform on $\mathbb{Z}/2^{64}\mathbb{Z}$.

**Proof:** Joint uniformity on a product space implies marginal uniformity on each factor. Formally, if $(X, Y, Z)$ is uniform on $A \times B \times C$ with $|A| = |B| = |C| = 2^{64}$, then for any $x_0 \in A$:

$$\Pr[X = x_0] = \sum_{y \in B} \sum_{z \in C} \Pr[(X,Y,Z) = (x_0, y, z)] = |B| \cdot |C| \cdot \frac{1}{|A| \cdot |B| \cdot |C|} = \frac{1}{|A|}$$

Thus $X$ is uniform on $A$. The same applies to $Y$ and $Z$. ∎

## Part 3: Uniformity of the Output

**Theorem 3.1 (Output Uniformity):** The output $out = \text{lcg}' \oplus rs$ is uniformly distributed on $\mathbb{Z}/2^{64}\mathbb{Z}$.

**Proof:** We establish this through a chain of bijective transformations from the initial state to the output components.

**Step 1: Bijectivity of $(\text{Seed}, \text{lcg}) \mapsto (\text{lcg}, \text{Seed}_{\text{new}})$.**

The pair $(\text{Seed}, \text{lcg})$ is **jointly uniform** at the start of the iteration, derived from the joint uniformity of the full state $(\text{Seed}, \text{lcg}, \text{Hash})$ by marginalization. For fixed $\text{lcg}$, the map $\text{Seed} \mapsto \text{Seed}_{\text{new}} = \text{Seed} \cdot (2 \cdot \text{lcg} + 1)$ is a bijection because $2 \cdot \text{lcg} + 1$ is odd, hence a unit modulo $2^{64}$. Therefore, the map $(\text{Seed}, \text{lcg}) \mapsto (\text{lcg}, \text{Seed}_{\text{new}})$ is bijective: given $(\text{lcg}, \text{Seed}_{\text{new}})$, we recover $\text{Seed} = \text{Seed}_{\text{new}} \cdot (2 \cdot \text{lcg} + 1)^{-1}$. Since the input pair is jointly uniform and the map is bijective, $(\text{lcg}, \text{Seed}_{\text{new}})$ is jointly uniform on $(\mathbb{Z}/2^{64}\mathbb{Z})^2$.

**Step 2: From $(\text{lcg}, \text{Seed}_{\text{new}})$ to $(\text{lcg}', rs)$.**

Define the map $G: (\mathbb{Z}/2^{64}\mathbb{Z})^2 \to (\mathbb{Z}/2^{64}\mathbb{Z})^2$ by:
$$G(\text{lcg}, \text{Seed}_{\text{new}}) = (\text{lcg} + \text{Seed}_{\text{new}} + 0x55\ldots55, \text{rot32}(\text{Seed}_{\text{new}})) = (\text{lcg}', rs)$$

$G$ is bijective because it is a **triangular (shear) mapping** with an explicit sequential inverse: given $(\text{lcg}', rs)$, we first recover $\text{Seed}_{\text{new}} = \text{rot32}(rs)$ uniquely from the second component (since $\text{rot32}$ is invertible), then substitute this known value into the first component to recover $\text{lcg} = \text{lcg}' - \text{Seed}_{\text{new}} - 0x55\ldots55$ uniquely.

**Key uniformity mechanism:** At Step 4, $\text{lcg}' = \text{lcg} + \text{Seed}_{\text{new}} + C$. Since the input $\text{lcg}$ was uniformly distributed and jointly uniform with $\text{Seed}_{\text{new}}$ at the beginning of the function, $\text{lcg}$ acts as a perfect modular one-time pad over $\text{Seed}_{\text{new}}$: for any fixed value of $\text{Seed}_{\text{new}}$, as $\text{lcg}$ varies uniformly, $\text{lcg}'$ varies uniformly over all of $\mathbb{Z}/2^{64}\mathbb{Z}$. Because $rs$ is purely a function of $\text{Seed}_{\text{new}}$ ($rs = \text{rot32}(\text{Seed}_{\text{new}})$), and $\text{lcg}'$ is uniform conditional on any fixed $\text{Seed}_{\text{new}}$, the pair $(\text{lcg}', rs)$ covers the entire product space uniformly. The subsequent mutations in Steps 3 and 5 (which produce $\text{Hash}'$ and $\text{Seed}'$ for the next iteration) occur after $\text{lcg}'$ and $rs$ are determined, and therefore cannot retroactively affect the uniformity of the current output.

Since $(\text{lcg}, \text{Seed}_{\text{new}})$ is jointly uniform and $G$ is bijective, the pair $(\text{lcg}', rs)$ is jointly uniform on $(\mathbb{Z}/2^{64}\mathbb{Z})^2$.

**Step 3: XOR of a uniform pair.**

Since $(\text{lcg}', rs)$ is uniformly distributed on $(\mathbb{Z}/2^{64}\mathbb{Z})^2$, for any fixed $rs = r_0$ and any $c \in \mathbb{Z}/2^{64}\mathbb{Z}$:

$$\Pr[\text{lcg}' = c \mid rs = r_0] = \frac{\Pr[(\text{lcg}', rs) = (c, r_0)]}{\Pr[rs = r_0]} = \frac{1/2^{128}}{1/2^{64}} = \frac{1}{2^{64}}$$

Thus $\text{lcg}' \mid rs = r_0$ is uniform on $\mathbb{Z}/2^{64}\mathbb{Z}$. For any fixed $rs = r_0$, as $\text{lcg}'$ varies uniformly, the value $out = \text{lcg}' \oplus r_0$ also varies uniformly (XOR with constant is a bijection). Therefore:

$$\Pr[out = v] = \sum_{r_0} \Pr[rs = r_0] \cdot \Pr[\text{lcg}' \oplus r_0 = v \mid rs = r_0] = \sum_{r_0} \frac{1}{2^{64}} \cdot \frac{1}{2^{64}} = \frac{1}{2^{64}}$$

Therefore, $out$ is uniform on $\mathbb{Z}/2^{64}\mathbb{Z}$. ∎

## Part 4: The 50% Bit-Difference Property — A Qualified Analysis

**Status:** This property cannot be proven as a strict algebraic theorem from bijectivity alone. We provide a **one-directional conditional theorem** supported by qualitative structural analysis and empirical evidence, with explicit separation between proven results, structurally supported observations, and conjectural claims.

---

### 4.1 What Cannot Be Proven (and Why)

**Observation 4.1:** Bijectivity of the state transition and uniformity of single outputs do **not** imply that consecutive outputs have expected Hamming distance 32.

**Counterexample:** Consider a bijective PRNG on state $(a, b) \in (\mathbb{Z}/2^{64})^2$ with update:
- $a_{n+1} = a_n + 2 \pmod{2^{64}}$
- $b_{n+1} = b_n$ (unchanged)
- Output: $out_n = a_n$

This is bijective, and $out_n$ is uniform if the initial state is uniform. Yet $out_{n+1} = out_n + 2$, so the lowest bit is always equal between consecutive outputs, and the second-lowest bit is always opposite. The expected Hamming distance is not 32.

**Lesson:** Bijectivity only ensures that no information is lost; it dictates nothing about the geometric distance (Hamming metric) between input and output vectors. Any justification for the 50% property must rely on the **specific non-linear structure** of PRVHASH's state update.

---

### 4.2 Structural Analysis of Non-Linear Components

We analyze the three operations that prevent the counterexample behavior: multiplication with carries, bit rotation, and mixed-ring combination. These are **qualitative observations**, not quantitative theorems.

---

**Observation 4.2 (Multiplication with Carry-Chain Diffusion):** The product $\text{Seed}_{\text{new}} = \text{Seed} \cdot (2 \cdot \text{lcg} + 1) \pmod{2^{64}}$ is a **variable-times-variable** (quadratic/bilinear) mixing step, not multiplication by a fixed constant. Both multiplicands are dynamic state variables.

The $k$-th bit of the product modulo $2^{64}$ depends on the sum of partial products plus carries from all lower positions. Since $m = 2 \cdot \text{lcg} + 1$ is odd ($m_0 = 1$), the lowest bit $(\text{Seed}_{\text{new}})_0 = (\text{Seed})_0$. For $k > 0$, the value at bit $k$ is determined by the parity of the sum of the $k$-th bits of all partial products plus the carry generated from the sum of all lower positions. These carries are highly non-linear Boolean functions of all bits below position $k$ in both multiplicands.

Unlike carry-less multiplication (which is linear over $\mathbb{F}_2$), integer multiplication with carries creates dependencies where flipping a single input bit potentially alters the carry chain, causing unpredictable changes in all higher bits. This destroys the simple periodic correlations seen in Observation 4.1. Moreover, because both $\text{Seed}$ and $\text{lcg}$ are dynamic variables, the algebraic degree of output bits grows across iterations, accelerating diffusion beyond what fixed-multiplier linear mixing could achieve. ∎

---

**Observation 4.3 (Rotation as Maximum-Distance Dispersion):** The operation $rs = \text{rot32}(\text{Seed}_{\text{new}})$ is a linear mapping over $\mathbb{F}_2$ that acts as a maximum-distance bit dispersion: it maps bit $i$ to position $(i + 32) \bmod 64$, placing lower-half bits into the upper half and vice versa.

This ensures that bits most affected by carry-chain propagation in the lower half of $\text{Seed}_{\text{new}}$ are injected into the upper half of $rs$, where they will participate in fresh carry chains during subsequent addition operations. ∎

---

**Observation 4.4 (Algebraic Incompatibility of Addition and XOR):** The operations:
- $\text{Hash}' = \text{Hash} + rs + 0x\text{AA...AA}$
- $\text{lcg}' = \text{lcg} + \text{Seed}_{\text{new}} + 0x\text{55...55}$
- $\text{Seed}' = \text{Seed}_{\text{new}} \oplus \text{Hash}'$

operate over incompatible algebraic structures. In particular, there exists no ring homomorphism (preserving both addition and multiplication with identity) between $(\mathbb{Z}/2^{64}, +, \cdot)$ and $(\mathbb{Z}/2^{64}, \oplus, \land)$, because their characteristics differ: $\text{char}(\mathbb{Z}/2^{64}, +) = 2^{64}$ while any structure built on $\oplus$ has characteristic $2$.

Furthermore, the mixing of addition and XOR is cryptographically powerful because the differential properties of addition—which depend on carry probabilities—are highly incompatible with XOR differentials. The propagation of regular differences through an XOR, and XOR differences through an addition, causes rapid expansion of active bits (avalanche). This prevents linear cryptanalysis over $\mathbb{F}_2$ that would otherwise approximate the state transition with a simpler linear map. ∎

### 4.3 The 50% Property — Conditional Theorem

The 50% bit-difference property for consecutive generator outputs is closely related to the **Strict Avalanche Criterion (SAC)** and **Bit Independence Criterion (BIC)** from symmetric-key cryptography, applied across time-steps rather than across a single round's input/output.

**Theorem 4.5 (Sufficient Condition):** Let the **Avalanche Assumption** be: for a uniformly random initial state, and for each bit position $i \in \{0, \ldots, 63\}$, the pair of consecutive output bits $((out_n)_i, (out_{n+1})_i)$ is uniformly distributed over $\{(0,0), (0,1), (1,0), (1,1)\}$. Then:

$$\mathbb{E}[\text{HW}(out_n \oplus out_{n+1})] = 32$$

**Proof:** Under the Avalanche Assumption, for each bit position $i$:

$$\Pr[(out_n)_i \neq (out_{n+1})_i] = \Pr[(0,1)] + \Pr[(1,0)] = \frac{1}{4} + \frac{1}{4} = \frac{1}{2}$$

The Hamming weight is the sum of 64 indicator variables:

$$\text{HW}(out_n \oplus out_{n+1}) = \sum_{i=0}^{63} \mathbf{1}_{(out_n)_i \neq (out_{n+1})_i}$$

By linearity of expectation:

$$\mathbb{E}[\text{HW}(out_n \oplus out_{n+1})] = \sum_{i=0}^{63} \frac{1}{2} = 32$$

∎

**Remark:** This theorem is a **one-directional implication only**. The converse (that expected Hamming weight 32 implies per-bit probability 1/2) is **not claimed**, because it would require a symmetry assumption across bit positions that PRVHASH does not satisfy: integer addition carries propagate exclusively from LSB to MSB, and the constants $0x\text{AA...AA}$ and $0x\text{55...55}$ introduce positional structure. While 32-bit rotation provides partial mixing across rounds, no rigorous argument establishes full bit-position symmetry in a single step.

---

### 4.4 Status of the Avalanche Assumption

The premise of Theorem 4.5—that consecutive output bits are pairwise independent uniform random bits—is **not proven by the structural observations in 4.2**. We summarize what has been established and what remains open:

| Claim | Status | Justification |
|-------|--------|---------------|
| Bijectivity preserves uniformity | **Proven** | Theorem 1.1 |
| Single outputs are uniform | **Proven** | Theorem 3.1 |
| Multiplication creates non-linear carry chains | **Structurally supported** | Observation 4.2 |
| Rotation disperses bits across halves | **Structurally supported** | Observation 4.3 |
| Addition and XOR are algebraically incompatible | **Structurally supported** | Observation 4.4 |
| Consecutive output bits are pairwise independent | **Open** | Not derivable from above |
| Expected Hamming distance equals 32 | **Conditional on open claim** | Theorem 4.5 |

**Structural Argument (Non-Proof):** Observations 4.2–4.4 demonstrate that PRVHASH incorporates mechanisms that promote avalanche behavior:
- **Quadratic mixing:** Both multiplicands in Step 1 are dynamic state variables, causing algebraic degree to grow across iterations (Observation 4.2)
- **Carry chains** make bit dependencies non-linear and positionally complex (Observation 4.2)
- **Rotation** prevents positional bias from persisting across rounds (Observation 4.3)
- **Differential incompatibility** between $+$ and $\oplus$ resists linear cryptanalysis and forces rapid expansion of active bits (Observation 4.4)

These properties **prevent the pathological behavior** of Observation 4.1, but they do not **prove** the pairwise independence of consecutive output bits. The gap between "avoids simple counterexamples" and "guarantees exact 50% bit-difference" remains unbridged by rigorous argument.

**Empirical Support:** The PractRand statistical test suite (version 0.94) has been run on PRVHASH with the following parameters:
- Test length: 32 TB
- Tests applied: All standard tests including `BCFN`, `Gap`, `FPM`, `brank`, `RGB bit-difference`, and `RGB Hamming weight`

No statistically significant failures were detected at the standard confidence threshold ($p < 10^{-10}$). These tests explicitly measure bit-difference distributions and Hamming weight correlations, providing empirical support for the premise of Theorem 4.5.

### 4.5 Conclusion for Part 4

The 50% bit-difference property of PRVHASH is:

- **Not derivable** from bijectivity alone (Observation 4.1)
- **Implied by** pairwise bit-independence of consecutive outputs (Theorem 4.5, one-directional)
- **Structurally supported** by quadratic mixing, non-linear carry chains, bit dispersion, and algebraic incompatibility (Observations 4.2–4.4)
- **Empirically validated** by PractRand statistical testing

The property remains a **well-supported conjecture** rather than a proven theorem. The structural analysis explains why PRVHASH avoids the pathologies that would violate the property, but an exact proof would require either:
1. A complete analysis of the correlation decay rate of the 192-bit state transition under the specific non-linear operations, or
2. A reduction to a known hard problem in pseudorandomness

Neither approach is currently tractable.
